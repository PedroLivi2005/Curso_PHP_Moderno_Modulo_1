Regras para nomes identificadores:
caracter;
1° variáveis sempre começam com símbolo $
2° pode ser letra ou símbolo _
3° Aceita caracteres [a-z], [A-Z], [0-9] e [_]
4° Aceita caracteres da tabela ASCII a partir de 128
5° Aceita caracteres acentuados como á, õ, ç
6° A linguem é case sensitive em relação aos nomes
7° Nomes especias como $this não podem ser usados

Recomendações para dar nomes
1° Tente dar nomes claros e de fácil identificação
2° Evite nomes muito curtos ou muito longos
3° Defina um padrão e siga em todo o projeto
4° Para variáveis, dê preferência a letras minúsculas
5° Para constantes, dê preferência a letras maiúsculas
6° Use camelCase para métodos e atributos
7° Use SNAKE_CASE para nomear constantes
Ex:
	$nomeCompletoCliente = "Camel Case";
	$telefone_contato_fornecedor = "Snake Case";

Manipulando Strings com PHP / Vídeo 22

Quatro formatos de Strings

- double quoted = ""
interpreta o que foi escrito
. é o operador de concatenação
Ex1: $nome = "Gustavo";
echo "Olá $nome !"; -> Olá Gustavo !

Ex2: $sobrenome = "Guanabara \u{1F596}";	
echo $sobrenome"; -> Guanabara emoji

Ex3: const ESTADO = "RJ";
echo "Moro no ESTADO"; -> Moro no ESTADO

echo "Moro no " .ESTADO; -> Moro no RJ


- single quoted = ''
não há interpretação do conteúdo
Ex: $nome = "Gustavo";
echo "Olá $nome !"; -> Olá $nome !

Ex2: $sobrenome = 'Guanabara \u{1F596}';	
echo $sobrenome"; -> Guanabara \u{1F596}
